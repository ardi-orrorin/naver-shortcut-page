## 버전 관리

1. release/v<버전> 로 배포
2. 버전은 1.0.0 으로 시작

## 코드 수정

0. 코드 수정하면 테스트로 npm run lint르 실행
1. 코드 수정이 필요한 경우 브렌치 기준으로 <현재 브렌치>/fix-<수정할 내용> 브렌치를 생성 하고 코드를 수정,
   1-2. 단 현재 브렌치가 main 이면 dev/fix-<수정할 내용>/main 브렌치를 생성하고 코드를 수정,
   1-3. 이슈가 있을 경우는 <현재 브렌치>/fix-`<issue 번호>`로 생성 하도 코드 수정
1. 브렌치 병합은 개발자가 컴펌 후 직접 병합

## 브렌치 병합 요청처리 방법

1. 코드 수정 요청 후 브렌치 병합 요청할 경우 상위 브렌치로 병합, 메시지는 요청한 수정 내용을 기술
2. 예를 들어 dev/fix-`<issue 번호>`/fix-`<수정 내용>` 처럼 하위의 하위 브렌츠를 병합 요청시 한 단계 상위 브렌츠로 병합
